# ブログ管理画面 - 使用ガイド

Astroブログの記事を一元管理できる非公開の管理画面を実装しました。複数記事の選択・削除・一括編集が可能です。

## 🚀 アクセス方法

### 管理画面へのアクセス
```
http://localhost:4321/admin/blog-manager/
```

本番環境：
```
https://yoursite.com/admin/blog-manager/
```

## 🔐 ログイン

### デフォルトパスワード
```
admin123
```

**⚠️ 本番環境での変更方法:**

`src/pages/admin/blog-manager.astro` の以下の部分を編集：

```typescript
const password = 'admin123'; // ここを変更
```

または、環境変数で管理することを推奨：

```typescript
const password = import.meta.env.ADMIN_PASSWORD || 'admin123';
```

## 📋 機能説明

### 1. ログイン画面
- パスワードを入力してログイン
- ログイン情報はクッキーに保存されます
- セッションは永続的です（ブラウザを閉じるまで有効）

### 2. 記事一覧表示
管理画面に入ると、すべてのブログ記事が表示されます。

**表示項目:**
- ☑️ チェックボックス（複数選択用）
- タイトル
- スラッグ（記事の識別子）
- 日付
- ステータス（公開/下書き/ゴミ箱）
- カテゴリ

### 3. 記事の選択

#### 個別選択
- 各記事の横のチェックボックスをクリック

#### 全選択
- ヘッダーの「☑️」をクリックすると全件選択
- もう一度クリックするとすべて解除

#### 選択状況
- 画面下部に「○ 件選択中」と表示

### 4. 一括操作

**ステップ:**

1. 操作対象の記事を選択
2. 「一括操作を選択」ドロップダウンから操作を選択
3. 「適用」ボタンをクリック
4. 確認ダイアログで実行

**利用可能な操作:**

| 操作 | 説明 | 注意点 |
|------|------|--------|
| 公開にする | ステータスを「publish」に変更 | 公開ページに表示されます |
| 下書きにする | ステータスを「draft」に変更 | 公開ページに表示されません |
| ゴミ箱へ移動 | ステータスを「trash」に変更 | 記事は削除されません |
| 完全に削除 | JSONから完全削除 | **操作は取り消せません** |

### 5. 削除機能

⚠️ **「完全に削除」は取り返しのつきません**

実行前に：
1. 必要に応じてバックアップを取る
2. 本当に削除してもいいか確認する
3. 削除確認ダイアログで再度確認

## 📊 ステータスの意味

- **公開 (publish)**: ブログに表示される
- **下書き (draft)**: 公開ページに表示されず、URL直打ちでのアクセスも基本的に不可
- **ゴミ箱 (trash)**: 削除予定の記事（削除前に一時保管）

## 🔧 データ保存方法

すべての変更は JSON ファイルに直接保存されます：

```
src/data/wp-posts.json
```

**自動バックアップの推奨:**

定期的にこのファイルをバージョン管理システム（Git）にコミットすることをお勧めします：

```bash
git add src/data/wp-posts.json
git commit -m "backup: blog posts"
```

## ⚙️ 設定のカスタマイズ

### パスワード変更
`src/pages/admin/blog-manager.astro` を編集：

```typescript
// 行目: const password = 'admin123';
const password = 'your-new-password';
```

### 認証パスの変更（上級）
ファイルの以下の行を編集：

```typescript
const authToken = Astro.cookies.get('admin_auth')?.value;
```

これを環境変数に変更したい場合は、別途カスタマイズしてください。

## 🛡️ セキュリティに関する注意

### ⚠️ 本番環境での推奨事項

1. **HTTPS必須** - パスワードは必ずHTTPSで送信
2. **強力なパスワード** - 推測されにくいパスワードを使用
3. **環境変数を使用** - パスワードをコードに直書きしない
4. **アクセス制限** - ファイアウォール等でアクセスを制限
5. **定期的なバックアップ** - JSONファイルのバックアップを取る

### より高度なセキュリティ
本番環境では以下を検討：
- JWT認証の導入
- 複数ユーザー管理
- 操作ログの記録
- IP制限

## 🐛 トラブルシューティング

### Q: パスワードを忘れてしまった
A: `src/pages/admin/blog-manager.astro` でパスワードを確認または変更してください

### Q: 削除した記事を復元したい
A: Git履歴から復元できます：
```bash
git log --oneline src/data/wp-posts.json
git checkout [commit-hash] src/data/wp-posts.json
```

### Q: 大量の記事を一括で操作したい
A: 管理画面の「全選択」を使用するか、JSONファイルをテキストエディタで直接編集してください

### Q: 記事の他の情報（タイトル、内容など）も編集したい
A: 現在は削除・ステータス変更のみ対応しています。詳細編集が必要な場合：
- JSONファイルをテキストエディタで直接編集
- または、より高度な管理画面の実装をご検討ください

## 📞 サポート

問題が発生した場合は、以下を確認してください：
1. コンソールエラーを確認（開発者ツール）
2. ファイルの権限設定を確認
3. JSONファイルの形式が正しいか確認
